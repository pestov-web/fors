Цель — чистая, семантичная, доступная и производительная верстка, легко переносимая в шаблон 1С-Битрикс: Управление сайтом без ломки структуры.

## Главные принципы
- Семантика прежде всего. Используй правильные теги (header, nav, main, section, article, aside, footer, h1–h6, ul/ol, figure/figcaption, form, label, button, и т.д.).
- Адаптив по умолчанию. Mobile-first, резиновая сетка, clamp() для типографики, гибкие изображения.
- Доступность (a11y). Навигация с клавиатуры, фокус-стили, aria-* только при необходимости, контраст не ниже WCAG 2.1 AA.
- Производительность. Ленивая загрузка, актуальные форматы изображений (<picture> + avif/webp), минимизация DOM, отсутствие лишнего JS и зависимостей.
- Чистота интеграции с Битрикс. Никаких хардкодов путей/URL, никаких инлайн-скриптов и стилей, продуманные точки вставки для компонентов и инклюдов.
- Поддерживаемость. BEM-наименование, единый код-стайл, линтеры, небольшие, переиспользуемые блоки.
- Прогрессивное улучшение. Верстка должна работать без JS; JS — как улучшение.
- Безопасность. Никаких инлайн-SVG с небезопасным контентом, никаких небезопасных обработчиков событий в HTML (вроде onclick).

## Конвенции наименования и структура CSS/JS

**2.1 BEM + утилиты**
- Блоки: card, элементы: card__title, модификаторы: card--featured.
- JS-хуки не смешивать с BEM: использовать префикс js- на классах без стилей (например, js-tabs).
- Утилитарные классы — префикс u- (например, u-hidden, u-visually-hidden, u-text-center).

**2.2 CSS (без препроцессора на старте)**
- Единицы: rem/em для размеров, %/vh/vw — по назначению. Пиксели — только там, где необходимо.
- Размеры шрифтов и интерлиньяж: через clamp() и/или относительные единицы.
- Медиа-запросы: mobile-first. Брейкпоинты по содержимому, ориентиры:
- @media (min-width: 360px), 768px, 992px, 1200px, 1440px, 1920px.
- Сетка — CSS Grid/Flexbox, без сторонних фреймворков.
- Цвета и размеры — CSS-переменные в :root.
- Не использовать !important, кроме как для утилит и критических хаков (и то осознанно, с комментарием).

**2.3 JS**
- Чистый ES-модульный JS без jQuery.
- Инициализация по DOMContentLoaded.
- Никаких инлайн-обработчиков (onclick в HTML). Навешивать события через JS.
- Данные для JS — только через data-*.
- Избегать глобалей; каждый блок — модуль, инициируемый по селектору корневого блока.

## Семантика, доступность, типографика

- Единственный h1 на страницу (в области main).
- Заголовки не пропускать уровни.
- Ссылки — только для навигации; кнопки — для действий. Не путать.
- У всех интерактивных элементов должен быть видимый :focus (не убирать outline без адекватной замены).
- Формы: label связан с input через for/id. Полезные атрибуты: required, autocomplete, inputmode, pattern.
- Текстовые альтернативы: каждое значимое изображение — alt; декоративные — пустой alt.
- Контраст: минимум 4.5:1 для текста, 3:1 для крупного текста/иконок.
- lang="ru" в <html>, корректная кодировка и мета-теги.

## Изображения и иконки

picture с источниками AVIF/WebP + PNG/JPG fallback.
Пример: 
```html
<picture>
  <source type="image/avif" srcset="/assets/img/hero.avif" />
  <source type="image/webp" srcset="/assets/img/hero.webp" />
  <img src="/assets/img/hero.jpg" alt="…" loading="lazy" decoding="async" width="…" height="…" />
</picture>
```
- Обязательно width/height (или aspect-ratio) для предотвращения layout shift.
- Иконки — через SVG-спрайт (symbol-sprite) или инлайн-SVG. Для массовых иконок — спрайт.
- Не вставлять растровые иконки для UI.

## Таблицы, списки, модалки

- Таблицы — семантические: thead, tbody, th scope="col|row".
- Модальные окна: фокус-трап, aria-modal="true", закрытие по Esc, запрет прокрутки фона, возврат фокуса.
- Списки — ul/ol по смыслу, не использовать для сетки.

## Правила для интеграции в 1С-Битрикс

Предусмотреть точки для компонентов: места, куда позже встанут bitrix:news, bitrix:menu, bitrix:breadcrumb, bitrix:catalog.* и т.п. — помечать HTML-комментариями:
```html
<!-- bx:breadcrumb -->
<!-- bx:main-menu -->
<!-- bx:news-list -->
```

## Готовые частичные шаблоны
- `includes/header.php` — основная шапка с жёлтым фоном. Плейсхолдеры: `bx:include:site-logo`, `bx:main-menu`, `bx:search-title`, `bx:include:header-phone`, `bx:main-menu-mobile`.
- `includes/header-inner.php` — белая шапка для внутренних страниц, модификатор `site-header--inner`, лёгкая кнопка переключения темы (`action-btn--theme-light`), те же плейсхолдеры.
- `includes/footer.php` — футер с CTA и контактами. Плейсхолдеры: `bx:footer-menu-primary`, `bx:footer-menu-secondary`, `bx:footer-cta`, `bx:footer-contacts`, `bx:footer-legal`, `bx:footer-social`. CTA использует фон `assets/bg/button.svg`.
- `includes/steps.php` — сетка «Этапы обучения» (нумерованный список). Плейсхолдер для замены контента: `bx:steps-list`. Заголовок секции задаётся в шаблоне страницы.
- `includes/form_consult.php` — форма заявки на консультацию. Плейсхолдер формы: `bx:form-consult`, содержит поля `name`, `phone` и кнопку «Оставить заявку».
- `includes/categories_and_prices.php` — карточки категорий с ценами. Плейсхолдер: `bx:categories-slider`; верстается как свайп-слайдер (`.js-categories-slider`) с пагинацией и стрелками.
- `includes/breadcrumbs.php` — хлебные крошки с плейсхолдером `bx:breadcrumb-path`.
- `index.php` подключает `includes/header.php` и `includes/footer.php`. При необходимости замены шапки на внутренних страницах подключать `includes/header-inner.php`.

## Актуальные активы
- `assets/css/styles.css` — базовые стили + модификатор `site-header--inner`, кнопка CTA. `body` и `main` выстроены флексами, чтобы футер прижимался к низу.
- `assets/css/media.css` — адаптив для футера и шапки.
- `assets/icons/` — SVG-иконки интерфейса (логотип, карта, телефон, почта, соцсети). Использовать напрямую или собрать спрайт.
- `assets/bg/button.svg` — фон кнопки «Записаться на курс» в футере.

## Чистота кода и форматирование
- Prettier: обязательное форматирование перед коммитом.
- Конец файла — перевод строки, UTF-8 без BOM, LF.
- HTML: двойные кавычки в атрибутах, один атрибут — одно значение, порядок атрибутов: class, id, data-*, aria-*, остальные.
- Комментарии: краткие и по делу; нет «мусорных» комментариев.

## Примерная структура проекта

```bash
/assets
  /css
    styles.css           # базовые стили, переменные, утилиты, сетка, типографика
  /img/                  # изображения (авиф/вебп/джпег + исходники)
  /icons/
    sprite.svg           # svg-спрайт иконок
  /js
    app.js               # инициализация модулей, навешивание событий
/src
  index.html             # пример страницы, разметка без битриксовых вставок
  pages/                 # прочие страницы-макеты, если нужно
  partials/              # куски разметки (header/footer/блоки), используемые при сборке/копипасте
  blocks/                # переиспользуемые блоки (BEM): card, hero, tabs, modal...
.github/
  workflows/             # CI (проверки, интеграция с ИИ-агентом)
AGENTS.md
```
## Что нельзя делать

❌ Инлайновые стили/скрипты, обработчики событий в атрибутах.
❌ Хардкод доменов/полных путей.
❌ Подключение библиотек без обсуждения (jQuery, большие UI-фреймворки).
❌ Пустые/многочисленные див-обертки без смысла.
❌ Использование изображений для текста/иконок интерфейса.
❌ Срывать фокус-стили без замены.
❌ Непродуманные медиа-запросы «ради точки макета», а не ради контента.
