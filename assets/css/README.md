# Структура CSS

Стили проекта организованы по модульному принципу для удобства поддержки и масштабирования.

## Структура файлов

```
assets/css/
├── styles.css              # Главный файл с импортами
├── styles.css.backup       # Резервная копия оригинала
├── base.css                # Переменные, reset, утилиты, тёмная тема
├── layout.css              # Header, footer, общие layout-компоненты
├── components/             # Переиспользуемые компоненты
│   ├── mobile-menu.css     # Мобильное меню
│   ├── home-hero.css       # Героя главной страницы
│   ├── steps.css           # Этапы обучения
│   ├── form-consult.css    # Форма консультации
│   ├── categories.css      # Категории и цены (слайдер)
│   └── breadcrumbs.css     # Хлебные крошки
└── pages/                  # Стили для конкретных страниц
    ├── fast-courses.css    # Ускоренные курсы
    ├── about.css           # О нас
    ├── faq.css             # Вопросы-ответы
    ├── gallery.css         # Фотогалерея
    ├── news.css            # Список новостей
    ├── news-element.css    # Детальная страница новости
    ├── specsignal.css      # Спецсигналы
    ├── schedule.css        # Расписание
    ├── price.css           # Прайс
    ├── contacts.css        # Контакты
    ├── search.css          # Поиск
    ├── autodrome.css       # Автодром
    └── error.css           # Страница 404
```

## Принципы организации

### 1. base.css
Содержит фундаментальные стили:
- CSS-переменные (цвета, размеры, шрифты)
- Reset стили (`* { box-sizing: border-box }`)
- Базовые стили для `body`, `html`, `main`
- Утилиты (`.u-container`, `.u-visually-hidden`, `.u-text-center`)
- Общие секции (`.page-section`, `.section-title`)
- Тёмная тема (`.theme-dark`)

### 2. layout.css
Глобальные layout-компоненты:
- `.site-header` и его модификаторы (`--home`, `--inner`)
- `.site-footer` и все его компоненты
- Навигация (`.primary-nav`)
- Кнопки действий (`.action-btn`)

### 3. components/
Переиспользуемые UI-блоки, которые встречаются на разных страницах:
- Подключаются через `includes/*.php`
- Независимы от конкретной страницы
- Следуют BEM-нейминг

### 4. pages/
Стили специфичные для каждой страницы:
- Используются только на одной странице
- Могут включать уникальные секции и блоки
- Часто содержат адаптивные сетки и сложные layouts

## Как добавить новые стили

### Новый компонент
1. Создайте файл `components/название-компонента.css`
2. Добавьте импорт в `styles.css`:
```css
@import './components/название-компонента.css';
```

### Новая страница
1. Создайте файл `pages/название-страницы.css`
2. Добавьте импорт в `styles.css`:
```css
@import './pages/название-страницы.css';
```

### Новая CSS-переменная
Добавьте в `:root` в `base.css`:
```css
:root {
  --ваша-переменная: значение;
}
```

## Важные замечания

- **Не редактируйте** `styles.css.backup` — это резервная копия оригинала
- **Основной файл** `styles.css` теперь содержит только импорты
- **Порядок импортов важен**: base → layout → components → pages
- **BEM-нейминг обязателен** для всех новых компонентов
- **Mobile-first**: медиа-запросы используют `min-width`
- **CSS-переменные** используются везде где возможно

## Миграция на Bitrix

При переносе в Bitrix шаблон:
1. Скопируйте всю папку `assets/css/` в `/local/templates/forsazh/`
2. Обновите пути в `<link>` тегах:
```php
<link rel="stylesheet" href="<?= SITE_TEMPLATE_PATH ?>/assets/css/styles.css">
<link rel="stylesheet" href="<?= SITE_TEMPLATE_PATH ?>/assets/css/media.css">
```
3. Структура модулей сохраняется без изменений
