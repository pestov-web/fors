# Обзор CSS-стилей

## Общие сильные стороны
- Используются CSS-переменные для ключевых цветов и отступов, что облегчает поддержку темы и единый визуальный стиль (`:root`).
- Включены полезные утилиты: контейнер с адаптивными отступами и класс `u-visually-hidden` для доступности (`.u-container`, `.u-visually-hidden`).
- Для типографики и внутренних отступов широко применяется `clamp()`, что поддерживает адаптивность (`.page-section`, `.site-footer__cta-button`).
- Адаптивность организована mobile-first: значения по умолчанию заданы для мобильных устройств, а расширения выполняются через `@media (min-width: …)` (`assets/css/media.css`).

## Зоны для улучшения
1. **Неоднородность переменных**
   - Не все цвета вынесены в переменные: часть значений зашита хардкодом (`#ffffff`, `rgba(26, 26, 26, 0.75)` и др.). Это усложняет смену темы и глобальную корректировку палитры. Стоит расширить набор переменных или использовать семантические токены (`--color-text-muted`, `--color-surface` и т.п.).
2. **Дублирование правил**
   - Брейкпоинт с отключением главного меню прописан и в `styles.css`, и в `media.css`, что со временем ведёт к расхождению (`@media (max-width: 1024px) { .primary-nav { display: none; } }`). Лучше хранить правило в одном месте.
3. **Смешение единиц измерения**
   - Для отступов и размеров блоков всё ещё часто используются пиксели (`.fast-benefit-card`, `.burger`). В дизайн-системе, ориентированной на адаптив, стоит переходить на `rem`/`em` там, где нет жёсткой привязки к пиксельным размерам (особенно высоты/ширины интерактивных элементов).
4. **Состояния фокуса и доступность**
   - Кнопка `.action-btn` и ссылочные элементы шапки (`.site-header__phone`, `.primary-nav__link`) не имеют явного состояния `:focus-visible` или оно совпадает с `:hover`. Для управления с клавиатуры лучше задать заметный кастомный фокус (outline/box-shadow).
   - Утилита `outline: none;` внутри некоторых состояний (например, `.footer-nav__link:hover`) может скрыть стандартный фокус, если кастомный outline не сработает. Стоит избегать глобального отключения outline.
5. **Тёмная тема**
   - Модификатор `.theme-dark` переопределяет только фон и основной текст. Остальные переменные и фоновые изображения остаются светлыми (`background: #ffffff` у карточек). Тема работает не полностью — либо нужно расширить набор переменных, либо скрыть опцию, пока её нельзя корректно применить.
6. **Высоты/ширины без `min()`/`max()` ограничений**
   - Жёсткая высота `height: 372px` у `.fast-benefit-card` может привести к переполнению на устройствах с увеличенным масштабом. Лучше заменить на автоматическую высоту и контролировать выравнивание через `min-height`.
7. **Файл `media.css`**
   - В файле присутствуют повторяющиеся блоки с одинаковыми условиями (`@media (min-width: 768px)` объявлен дважды). Для читаемости можно сгруппировать правила под одним медиазапросом.

## Рекомендации по дальнейшему развитию
- Ввести дизайн-токены для оттенков текста/фона и использовать их во всех блоках.
- Настроить линтер (Stylelint) с правилами на запрет `outline: none` и использование единиц.
- Проверить работу `prefers-reduced-motion`: анимации наведения можно отключать или ослаблять для пользователей с ограничениями.
- Подготовить таблицу слоёв (например, через `@layer`) для структурирования базовых, компонентных и утилитарных стилей.
