# SEO-аудит HTML-разметки

## Общая оценка
Разметка аккуратная, с чётко выстроенной иерархией секций и использованием семантических тегов. Тем не менее, с точки зрения современного SEO остаются пробелы: не хватает базовых мета-тегов, микроразметки и более продуманной структуры заголовков на главной странице. Ниже перечислены сильные стороны и зоны роста.

## Что сделано хорошо
- **Семантическая структура.** Используются `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, а также списки `<ol>/<ul>` и `<table>` по назначению, что помогает поисковым роботам понимать структуру страницы (`index.php`, `fast_courses.php`).
- **Навигационные блоки с `aria-label`.** Основное меню, мобильное меню и хлебные крошки снабжены понятными атрибутами `aria-label`, что косвенно поддерживает SEO через улучшение доступности (`includes/header.php`, `includes/header-inner.php`, `includes/breadcrumbs.php`).
- **Правильное использование заголовков внутри блоков.** У карточек категорий, преимуществ и этапов используются `<h3>` — это помогает формировать насыщенные ключевыми словами подзаголовки, не нарушая иерархию (`includes/categories_and_prices.php`, `includes/steps.php`, `fast_courses.php`).
- **Табличные данные размечены таблицей.** Блок с ценами использует `<table>` с `scope="col/row"`, что облегчает парсинг структурированных данных (`fast_courses.php`).

## Основные проблемы
1. **Отсутствие ключевых мета-тегов.** В `<head>` нет `meta name="description"`, `meta name="robots"`, `link rel="canonical"`, Open Graph (`og:*`) и `twitter:*` тегов. Это снижает CTR в поисковой выдаче и может приводить к дублированию страниц (`index.php`, `fast_courses.php`).
2. **Нет глобальных структурированных данных.** Нигде не подключена микроразметка (schema.org) для организации, хлебных крошек, курсов или FAQ. Это упускает возможность обогащённых сниппетов.
3. **На главной странице отсутствует `<h1>`.** В `index.php` главный контент начинается сразу с `<section>` и `<h2>` из подключаемых инклюдов, из-за чего страница теряет главный заголовок, важный для SEO и доступности.
4. **Повторяющиеся идентификаторы при подключении блоков.** Идентификаторы `categories-title`, `consult-title` и другие из подключаемых секций повторяются при повторном использовании инклюда на одной странице (например, `fast_courses.php` повторно подключает `includes/categories_and_prices.php` и `includes/form_consult.php`). Это может создавать конфликт `id` и мешать навигации по `aria-labelledby`.

### Почему дублирующиеся `id` — проблема

`id` в HTML обязан быть уникальным в пределах страницы. Когда одна и та же секция подключается несколько раз и у неё захардкожены одинаковые `id`, возникают сразу несколько рисков:

- **Нарушение связи с `aria-labelledby` / `aria-describedby`.** Экранные дикторы ищут первый элемент с указанным `id`. Если таких элементов два, ассистивные технологии возьмут текст из первого найденного экземпляра или вовсе не смогут определить правильную подпись. Например, у блока `includes/form_consult.php` заголовок «Оставьте заявку» связан с формой через `aria-labelledby="consult-title"`, но при повторном подключении диктор будет озвучивать только самый первый заголовок на странице.
- **Проблемы с внутренней навигацией и якорными ссылками.** Ссылки вида `<a href="#categories-title">` будут всегда прокручивать страницу к первому совпадению. Пользователь, кликнувший якорь во второй секции, попадёт к блоку из первого подключения.
- **Сбой в работе JS-компонентов.** Любой скрипт, который пытается получить элемент по `document.getElementById('consult-title')`, вернёт только первый экземпляр, из‑за чего обработчики событий инициализируются не на всех блоках.

Чтобы избежать конфликтов, при множественном использовании инклюдов нужно передавать уникальный суффикс `id` через параметры компонента/инклюда или использовать шаблон типа `id="consult-title-<?= $arParams['FORM_ID']; ?>"`. Альтернативно можно отказаться от `aria-labelledby` в пользу локальных заголовков внутри блока, не завязанных на `id`.
5. **Ссылки без пояснительного текста.** В футере есть ссылки «Правовая информация Форсаж/Сигнал», оформленные простым текстом в `<p>` без `<a>`. Это упущенная возможность добавить полезные внутренние ссылки (`includes/footer.php`).
6. **Кнопки вместо ссылок для навигационных действий.** CTA «Записаться на курс» в карточках категорий и `site-footer__cta-button` реализованы через `<button>` или `<a>` без `href` на реальную посадочную страницу. Для SEO важны кликабельные ссылки на ключевые конверсии (`includes/categories_and_prices.php`, `includes/footer.php`).
7. **Повторяющиеся заголовки H2 на разных страницах без уточняющего контента.** Например, «Категории и цены» встречаются и на главной, и на странице ускоренных курсов без дополнительных уточнений. Это допустимо, но стоит продумать уникальность заголовков и текстов для разных посадочных страниц.

## Рекомендации по улучшению
1. Добавить в `<head>` базовые SEO-мета-теги: `description`, `robots`, `canonical`, Open Graph и Twitter-карточку. Для проекта на Битрикс предусмотреть плейсхолдеры под управление из настроек.
2. Внедрить структурированные данные schema.org: как минимум `Organization` в шапке/футере, `BreadcrumbList` для крошек и `Course`/`Product` для предложений категорий.
3. Обеспечить наличие одного `<h1>` на каждой странице. На главной странице можно вынести заголовок героя или основного предложения в шаблон до подключаемых секций.
4. Сделать `id` уникальными: передавать значения через параметры инклюда или использовать префиксы, чтобы повторное подключение одного блока не создавало дублей.
5. Превратить CTA в полноценные ссылки (`<a href="…">`) на соответствующие формы или лендинги, чтобы поисковые системы учитывали внутренние переходы.
6. Добавить реальные ссылочные элементы в футере для юридической информации, социальных сетей с атрибутами `rel="noopener nofollow"` при необходимости и осмысленными анкорами.
7. Проработать уникальность контента и заголовков на внутренних страницах: добавить уточняющие подзаголовки, описания преимуществ, FAQ — это повысит релевантность по разным запросам.
8. Рассмотреть добавление `lang`-атрибутов на блоки с иностранными словами/аббревиатурами и использование `title`/`meta` с локальными ключевыми фразами для каждой страницы.

## Следующие шаги
1. Согласовать перечень обязательных SEO-тегов и внедрить их в базовый шаблон `head` с возможностью управления из настроек Битрикс.
2. Разработать и подключить микроразметку (JSON-LD или Microdata) для организации, хлебных крошек и карточек курсов.
3. Доработать шаблон главной страницы: добавить блок героя с `<h1>` и уникальным описанием, усилить текст ключевыми запросами.
4. Перепроверить подключаемые инклюды на предмет уникальности `id` и семантики при множественном использовании.
5. Обновить CTA и юридические ссылки, чтобы они вели на целевые страницы и помогали строить внутреннюю ссылочную структуру.
